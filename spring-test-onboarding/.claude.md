# spring-test-onboarding (TestCraft) — 모듈 규칙

> ⚠️ 이 파일은 이 모듈 작업 시 AI 어시스턴트가 **절대적으로** 준수해야 하는 규칙입니다.
> 루트 `.claude.md`의 공통 규칙(상호 참조 금지, Java 1.8 문법, 로깅, 커밋 규칙)을 상속합니다.

## 🎯 모듈 목적

테스트 코드를 한 번도 작성해본 적 없는 SpringBoot 초보자가 **일주일(하루 1~2시간, 자기주도)** 안에
JUnit5 + SpringBootTest 기본기를 갖추도록 하는 **학습용 모듈**입니다.

- 커리큘럼: `docs/test/curriculum/00-TestCraft-Curriculum.md`
- Step별 교육 문서: `docs/test/education/FOR-Test-StepNN.md`

## 📐 example / exercise / answer 3종 규약 (MUST)

| 패키지 | 클래스명 | @Disabled | 빌드 시 |
|--------|---------|-----------|---------|
| `example` | `{대상}Test` | 없음 | 실행·통과 필수 |
| `exercise` | `{대상}ExerciseTest` | **클래스 레벨 필수** (`@Disabled("과제: docs/test/education/FOR-Test-StepNN.md 참고")`) | 컴파일만, skipped |
| `answer` | `{대상}AnswerTest` | 없음 | 실행·통과 필수 |

- exercise 메서드는 **시그니처 + given/when/then 주석 골격 + `// TODO` 한국어 지시문**만 작성
- exercise와 answer는 1:1 대응 (메서드명 동일)
- 어떤 커밋에서도 `gradlew :spring-test-onboarding:test`는 **그린**이어야 함

## 🧪 테스트 작성 규칙

- 테스트 메서드명: `{대상}_{시나리오}_{예상결과}` 한글 허용 (예: `signup_중복아이디_예외발생`)
- `@DisplayName` 병기 권장
- 단언은 **AssertJ만** 사용 (`assertThat`, `assertThatThrownBy`) — JUnit `Assertions.assertEquals` 금지
- AAA 패턴 주석 필수: `// given` `// when` `// then`
- 모든 example/answer 테스트에는 **왜 이 테스트가 필요한지** 한국어 Javadoc 필수
- Mockito는 BDD 스타일 (`given(...).willReturn(...)`) 우선

## 🔒 Security 규칙 (MUST)

- `WebSecurityConfigurerAdapter` **사용 금지** (deprecated) — `SecurityFilterChain` @Bean 방식만
- 인터넷의 구식 예제(extends WebSecurityConfigurerAdapter)를 절대 따라하지 말 것
- 소유자 검증(작성자 본인만 수정/삭제)은 Service 계층 책임 — Security 설정에 넣지 않음

## 🗄️ SQL 규칙 (MS-SQL 스타일)

- 페이징: `OFFSET n ROWS FETCH NEXT m ROWS ONLY` (LIMIT 금지)
- PK 채번: `BIGINT IDENTITY` + MyBatis `useGeneratedKeys="true"`
- 날짜 기본값: `GETDATE()`
- H2 MODE=MSSQLServer에서 안 되는 T-SQL(OUTPUT 절, 프로시저, TOP WITH TIES)은 사용 금지하고 문서에 차이를 기록

## 📊 커밋 규칙

```
✨ feat: [Test/Step N] 제목

상세 설명

학습 포인트:
- 포인트 1
- 포인트 2
```

- Step 단위 커밋 = 학습 단위 (해당 Step의 프로덕션 코드 + 테스트 3종 + 교육 문서를 함께 커밋)
- 매 커밋 전 `.\gradlew :spring-test-onboarding:test` 그린 확인
